COMMON BACKTEST ERROR PATTERNS AND FIXES:

1. **Empty DataFrame from yfinance**:
   - Cause: Ticker doesn't exist, API rate limit, or date range issue
   - Fix: Add validation after data fetch, use try-except, implement retry logic
   ```python
   data = yf.download(tickers, start=start, end=end)
   if data.empty:
       logger.warning(f"No data returned for {tickers}. Check ticker validity and date range.")
       # Fall back to available tickers
   ```

2. **NaN propagation in signals**:
   - Cause: Missing price data -> NaN returns -> NaN signals -> empty portfolios
   - Fix: Add minimum observation checks, fill or drop NaN explicitly
   ```python
   valid_count = returns.rolling(window=lookback).count()
   signal = signal.where(valid_count >= min_observations)
   ```

3. **Date index alignment**:
   - Cause: Different data sources have different trading calendars
   - Fix: Use pd.DataFrame.reindex with appropriate method
   ```python
   aligned = data.reindex(reference_dates, method='ffill')
   ```

4. **Timezone issues**:
   - Cause: Some data sources return timezone-aware dates, others naive
   - Fix: Normalize all dates to timezone-naive or a single timezone
   ```python
   df.index = df.index.tz_localize(None)  # Remove timezone
   ```

5. **Division by zero in normalization**:
   - Cause: All assets have the same signal value (std = 0) at some date
   - Fix: Check for zero std before dividing
   ```python
   std = signal.std(axis=1)
   std = std.replace(0, np.nan)  # Avoid division by zero
   z_score = signal.sub(signal.mean(axis=1), axis=0).div(std, axis=0)
   ```

6. **KeyError in portfolio sorting**:
   - Cause: Expected column not present after filtering removes all assets
   - Fix: Add defensive checks after filtering
   ```python
   filtered = data[data['market_cap'] > threshold]
   if filtered.empty:
       logger.warning(f"No assets passed filter for date {date}")
       continue  # Skip this rebalancing period
   ```

7. **Memory error with large datasets**:
   - Cause: Loading full history for too many tickers at daily frequency
   - Fix: Process in chunks, use monthly data where daily isn't needed
   ```python
   # Process in chunks of 100 tickers
   for chunk in np.array_split(tickers, max(1, len(tickers) // 100)):
       data_chunk = fetch_data(chunk)
   ```

8. **API rate limit errors**:
   - Cause: Too many requests to yfinance or other data providers
   - Fix: Add delays, implement caching, batch requests
   ```python
   import time
   time.sleep(1)  # Rate limiting
   # Or use caching:
   cache_path = f"data/cache/{ticker}_{start}_{end}.parquet"
   if os.path.exists(cache_path):
       return pd.read_parquet(cache_path)
   ```

DEBUG GUIDELINES:
1. NEVER suggest a fix that introduces look-ahead bias, even if it "solves" the error.
2. NEVER suggest hardcoding values as a fix. Always reference config.
3. Prefer defensive fixes that handle edge cases gracefully rather than just catching and suppressing errors.
4. If the fix requires changing methodology (not just implementation), flag this clearly as a paper fidelity concern.
5. Always suggest adding a test that would catch this error in regression testing.
6. Consider whether the error reveals a deeper data quality issue that affects the backtest's validity.
