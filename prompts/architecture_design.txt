ARCHITECTURE DESIGN GUIDELINES:
1. Each module should have a single, clear responsibility. Data loading is separate from signal computation.
2. Config must be the SINGLE source of truth for all parameters. No magic numbers in code.
3. The data flow should be strictly unidirectional: data -> signals -> portfolios -> analysis -> visualization.
4. All date-indexed DataFrames should use pandas DatetimeIndex for consistent time alignment.
5. Signal DataFrames and weight DataFrames should have identical structure: DatetimeIndex rows, asset columns.
6. The architecture must make look-ahead bias prevention structurally obvious (e.g., signal computation functions take only historical data).
7. Transaction costs should be computed separately and subtracted, making it easy to report gross vs. net results.
8. The design should support easy extension: adding new signals, alternative weighting schemes, or additional metrics.
