PLANNING GUIDELINES:
1. Every configurable parameter from the paper MUST appear in the config module. Never hardcode strategy parameters.
2. The data pipeline must ensure no look-ahead bias at every step. Data available on date T should only reflect information known before T.
3. Signal computation should be fully vectorized using pandas/numpy for performance.
4. Portfolio construction must respect the exact methodology in the paper (sorting, breakpoints, weighting).
5. Transaction costs should be modular - easy to toggle on/off or change assumptions.
6. All performance metrics should match the paper's methodology for fair comparison.
7. The plan should be implementable using freely available data (yfinance) as a fallback even if the paper uses proprietary data.
8. Include sufficient logging and intermediate output for debugging signal/portfolio construction issues.
