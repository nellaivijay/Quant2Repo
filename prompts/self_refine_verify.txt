VERIFICATION GUIDELINES:

For STRATEGY EXTRACTION artifacts:
- Verify all equations are correctly transcribed from the paper
- Check that all parameters have correct values
- Ensure no signals or methodology steps are missing
- Verify asset class, universe, and data requirements are complete

For BACKTEST PLAN artifacts:
- Verify the plan covers all aspects of the paper's methodology
- Check that the data pipeline handles all required data types
- Ensure signal generation steps match the paper exactly
- Verify portfolio construction rules are complete and correct

For ARCHITECTURE DESIGN artifacts:
- Verify all necessary modules are included
- Check that module interfaces are well-defined
- Ensure the data flow prevents look-ahead bias structurally
- Verify that config captures all tunable parameters

For GENERATED CODE artifacts:
- Check every formula against the paper
- Verify no look-ahead bias in signal construction
- Check that all parameters come from config (no hardcoded values)
- Verify proper handling of missing data and edge cases
- Check return computation methodology
- Verify rebalancing timing and lags

For VALIDATION REPORT artifacts:
- Check that the validation is thorough
- Verify that identified issues are real (not false positives)
- Check if the validation missed any important issues
- Verify scoring is calibrated appropriately

SEVERITY DEFINITIONS:
- CRITICAL: Will produce meaningfully wrong backtest results. Must fix immediately.
- MAJOR: Will noticeably affect backtest accuracy. Should fix before trusting results.
- MINOR: Small inaccuracy or code quality issue. Fix when convenient.
- SUGGESTION: Enhancement that would improve the artifact but is not necessary for correctness.
