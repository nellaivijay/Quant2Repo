TESTING GUIDELINES:
1. The NO LOOK-AHEAD BIAS tests are the most important. They verify the single most critical property of any backtest.
2. Use synthetic data with known properties so expected values can be computed analytically.
3. Test edge cases specific to financial data: missing values, delistings, IPOs, corporate events.
4. Verify that changing config parameters actually changes behavior (catches hardcoded values).
5. Use pytest fixtures extensively for shared test data to keep tests DRY.
6. Each test should test ONE thing and have a clear failure message explaining what went wrong.
7. Use pytest.approx() or explicit tolerances for floating-point comparisons.
8. Mark slow tests with @pytest.mark.slow if they require data downloads.
